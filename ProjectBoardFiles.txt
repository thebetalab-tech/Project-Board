Shared Files & Configuration3.1  Web.config  (Root)The root Web.config holds the SQL Server connection string, forms authentication settings, and session timeout. This file is read by DBHelper.cs on every request.Key SettingWhat It Does<connectionStrings>Stores the ProjectBoardDB connection string used by DBHelper.cs<authentication mode="Forms">Enables Forms Auth — Login.aspx is the loginUrl<authorization>Root level: denies all unauthenticated users, redirects to Login.aspx<sessionState timeout>Sets session idle timeout (recommend 30 min for college use)<customErrors mode>Set to RemoteOnly so detailed errors show in VS, generic errors in production3.2  Sub-folder web.config files  (/Student/, /Faculty/, /Admin/)Each role folder has its own web.config that restricts access using ASP.NET location-based authorization. This means even if someone guesses a URL, they cannot access a page outside their role.Folderweb.config Rule/Student/allow users="?" only if Role=Student; deny all others/Faculty/allow users="?" only if Role=Faculty; deny all others/Admin/allow users="?" only if Role=Admin; deny all others3.3  Site.MasterThe master page provides the shared navigation bar, header logo, and footer for all pages. Every .aspx page inherits from this master page using MasterPageFile="~/Site.Master" in its @Page directive.SectionContentNavigation BarLinks change based on Session["Role"]: Student sees Group/Project links, Faculty sees their dashboard, Admin sees management linksNotification BellShows unread count badge by calling NotificationDAL.GetUnreadCount(userId) on master page loadContentPlaceHolderThe main <asp:ContentPlaceHolder ID="MainContent"> where each .aspx page injects its contentFooterStatic footer with project name, semester, and guide name4.
